# BFSI AI Call Center Assistant

A lightweight, locally-running AI assistant for Banking, Financial Services, and Insurance (BFSI) call center queries. Built using a fine-tuned Small Language Model (SLM), dataset-based similarity matching, and a Retrieval-Augmented Generation (RAG) layer. The system prioritizes fast, accurate, and compliant responses while running entirely on local hardware — ensuring data privacy and zero dependency on external APIs.

---

## Architecture

```
User Query
    │
    ▼
Tier 1 — Dataset Similarity Match
    │  (semantic search over 170+ curated BFSI Q&A pairs)
    │  if confidence >= 0.60 → return stored response instantly
    │
    ▼
Tier 3 — RAG Layer (for policy/regulation keywords)
    │  (ChromaDB vector search over banking_policies.txt + loan_guidelines.txt)
    │  retrieves relevant context → model generates grounded response
    │
    ▼
Tier 2 — Fine-Tuned SLM
    │  (TinyLlama 1.1B fine-tuned with LoRA on BFSI dataset)
    │  generates response for novel queries
    │
    ▼
Guardrails
    (out-of-domain rejection, blocked keywords, no fake financial figures)
```

## Project Structure

```
BFSI/
├── dataset/
│   └── bfsi_alpaca.json          # 200+ curated BFSI Q&A samples (Alpaca format)
├── knowledge_base/
│   ├── banking_policies.txt      # RBI guidelines, account policies, insurance rules
│   └── loan_guidelines.txt       # Loan types, EMI rules, CIBIL, interest rates
├── models/
│   └── phi2_bfsi_slm/            # Fine-tuned LoRA weights (generated by 02_finetune.py)
├── chroma_db/                    # Vector index for RAG (generated by 03_build_rag.py)
├── 02_finetune.py                # Fine-tuning script (TinyLlama + LoRA)
├── 03_build_rag.py               # Builds ChromaDB index from knowledge base
├── 04_app.py                     # Main Gradio demo app
├── requirements.txt
└── README.md
```

## Setup Instructions

### Step 1 — Install PyTorch with CUDA
```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### Step 2 — Install all dependencies
```bash
pip install -r requirements.txt
```

### Step 3 — Run the app
```bash
python 04_app.py
```

Open your browser at **http://127.0.0.1:7860**

## First Time Setup 

If the fine-tuned model or RAG index are not included, run these first:

```bash
# Step 1: Fine-tune the model (requires GPU, takes ~5-10 mins)
python 02_finetune.py

# Step 2: Build the RAG knowledge index
python 03_build_rag.py

# Step 3: Run the app
python 04_app.py
```

## Tech Stack used

- **Language:** Python 3.10+
- **Base Model:** TinyLlama 1.1B Chat
- **Fine-Tuning:** HuggingFace Transformers + PEFT (LoRA)
- **Quantization:** BitsAndBytes 4-bit (NF4)
- **RAG:** ChromaDB + sentence-transformers
- **Embeddings:** all-MiniLM-L6-v2
- **Demo UI:** Gradio
- **GPU:** NVIDIA CUDA (tested on RTX 4050)

---

## Hardware Requirements

- **Recommended:** 8GB+ RAM, 4GB+ GPU VRAM
- **Optimal:** 16GB+ RAM, 6GB+ GPU VRAM
- **CPU only:** Works but will be very slow

---

## Guardrails and security

- Queries with confidence below 0.25 are rejected as out-of-domain
- Blocked keywords (hack, launder, bypass, etc.) are rejected immediately
- No financial figures are generated without grounding in the knowledge base
- All processing is local — no data leaves the machine

---

## Supported Query Types

- Loan eligibility and application status
- EMI calculation and repayment schedules
- Interest rate information (fixed, floating, MCLR, repo rate)
- Payment and transaction queries
- Account and KYC support
- Insurance queries (health, motor, life)
- RBI guidelines and banking regulations
